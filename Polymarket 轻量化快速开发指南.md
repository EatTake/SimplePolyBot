这是一份基于 EOA（外部拥有账户）和 CLOB Off-Chain API 的 Polymarket 轻量化快速开发指南，专为 Vultr 墨西哥城服务器与 T-100 动量策略量身定制。

### Polymarket 轻量化自动化交易开发指南

#### 1. 基础设施与前置环境模块

为了实现轻量化与快速开发，系统将摒弃复杂的 Gnosis Safe 代理多签钱包，直接使用 EOA 钱包进行交互。

- **依赖安装：** 强烈建议使用官方提供的 Python SDK 进行开发。可通过执行 `pip install py-clob-client web3` 进行安装 。

  <br />
- **资金准备：** 在 Polygon 主网上准备一个独立的 EOA 钱包（即直接通过私钥控制的钱包，如 MetaMask）。由于使用纯 EOA 模式，你必须在钱包中存入 USDC.e 作为购买合约的本金，同时还需要存入少量的 POL/MATIC 代币，用于支付最终赎回清算时的底层网络 Gas 费 。

  <br />
- **网络防屏蔽策略：** Polymarket 的核心 API 受到 Cloudflare 的严格防护，而 Vultr 数据中心的 IP 极易触发 403 Forbidden 拦截 。为了确保系统稳定，建议建立持久化的 WebSocket 长连接来获取行情源，以此绕过高频 REST API 轮询带来的封控风险。如果必须高频调用 REST 接口，需考虑集成指纹伪装工具（如 `curl-impersonate`）或使用住宅代理 。

  <br />

#### 2. 账户认证与初始化模块

EOA 模式极大简化了代码层面的授权复杂性。通过私钥，SDK 会在本地安全地完成 L1 签名，并派生出用于高频下单的 L2 API 凭证 。

<br />

- **客户端初始化：** 在实例化 `ClobClient` 时，必须强制指定 `signature_type=0`（这代表 EOA 签名模式），并将 `funder` 参数显式设置为你钱包的公开地址 。这样可以直接建立鉴权，免去代理钱包的部署与授权调用。

  <br />

#### 3. 行情数据与动量分析模块

为了兼顾高灵敏度与结算权威性，数据层必须实行“双轨制”获取：

- **高频动量源 (K值计算)：** 单独接入币安 (Binance) 等高流动性 CEX 的 WebSocket 现货深度流。在本地内存中维护一个 180 秒的滚动价格队列，利用普通最小二乘法 (OLS) 持续进行线性回归拟合，得出的斜率 $K$ 即为最灵敏的波动子因子。
- **绝对结算基准源：** 连接 Polymarket 的实时数据套接字 (RTDS)，端点为 `wss://ws-live-data.polymarket.com` 。订阅 `crypto_prices` 主题，提取 Chainlink 官方提供的 BTC/USD 预言机实时数据 。这是最终决定五分钟市场胜负的“裁判价格”，必须用它与 `start_price` 进行差额比对。

  <br />

#### 4. 交易策略与动态安全垫模块

在距离市场周期结束还有 100 秒时（T-100），策略引擎从静默状态唤醒，按秒级执行逻辑：

- **双层安全垫计算：**

  $\text{安全垫总厚度} = \text{Base Cushion} + (\alpha \times |K| \times \sqrt{\text{Time Remaining}})$

  当判断为下注 DOWN 时，要求 Chainlink 的实时价格低于 `start_price`，且差额绝对值必须严格大于安全垫总厚度，方可生成买入信号。
- **动态 Taker Fee 与上限价格修正：** Polymarket 对加密货币市场收取动态吃单费，基准费率为 0.072，且在概率为 50/50 时达到最高 1.80% 的手续费峰值 。 单次交易费用公式需硬编码至期望计算中：$\text{Fee} = C \times 0.072 \times p \times (1 - p)$ 结合剩余时间，动态查表设定最高买入限价。例如 T-90 秒时风险尚存，最高出价设为 0.75 美元；T-10 秒时确定性极高，上限放宽至 0.90 美元。

  <br />

#### 5. 链下极速执行模块 (订单发送)

由于网络本身存在 60ms 左右的物理延迟，执行模块的订单结构必须具备极强的防反噬能力。

- **致命订单类型：** 绝对禁止发送 GTC（一直有效）订单。你必须将订单参数设定为  `FAK` (部分成交并取消剩余) 。如果在网络传输这几十毫秒内，盘口流动性被抽干导致目标价格失效，匹配引擎会自动销毁该订单，确保你不会在剧烈反转时成为接盘方。

  <br />
- **EIP-712 构建：** 直接调用 SDK 的 `create_and_post_order`，只需传入目标 Token ID、设定的最高买价、份额数量以及价格精度（`tickSize` 通常为 "0.01"） 。SDK 会在本地完成哈希运算并发送至链下撮合引擎。

  <br />

#### 6. 链上异步获利清算模块

五分钟市场一旦由预言机裁决完毕，持仓便固化为确定性价值。由于使用 EOA，你可以绕过繁琐的 Relayer 多签转发，直接与智能合约通信。

- **异步解耦：** 构建一个独立的 Python 定时任务（例如每 10 分钟运行一次），不要让它阻塞主交易线程。查询所有已完结且获胜的市场（Condition ID）。
- **直接交互 CTF 合约：** 实例化 Polygon 上的条件代币框架 (CTF) 智能合约，地址为 `0x4D97DCd97eC945f40cF65F87097ACe5EA0476045`。
- **一键销毁提取：** 构建交易调用 `redeemPositions` 函数 。传入固定的 USDC.e 地址（`0x2791Bca1f2de4661ED88A30C99A7a9449Aa84174`）、全零的 `parentCollectionId`（`0x0000...0000`）、市场的 `conditionId`，以及代表所有潜在结果组合的索引集 `[1, 2]` 。

  <br />
- 交易上链后，智能合约会自动销毁你 EOA 钱包内的获胜份额（Losing 份额同样被销毁且价值为 0），并将按每份 1.00 美元计算的 USDC.e 即刻划拨至你的可用资金池 。
