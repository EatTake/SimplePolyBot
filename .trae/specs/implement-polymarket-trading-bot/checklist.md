# Checklist

本文档用于验证 V1.0 版本是否满足所有功能需求与非功能性需求。每个检查项对应 spec.md 中的一个或多个需求场景。

## 基础设施验证

### Redis 服务
- [ ] Redis 服务器成功安装并运行
- [ ] Redis 默认端口 6379 可访问
- [ ] Redis Pub/Sub 功能正常工作
- [ ] shared/redis_client.py 能够成功连接 Redis

### 配置管理
- [ ] 系统能够成功从 .env 文件加载私钥、API 凭证、Redis 连接信息
- [ ] 系统能够成功从 config/settings.yaml 加载策略参数
- [ ] 系统能够验证所有必需配置项已设置
- [ ] 配置缺失时系统能够提供清晰的错误提示

### 日志系统
- [ ] 日志能够同时输出至控制台与文件
- [ ] 日志格式为结构化 JSON，包含时间戳、模块名称、操作类型、关键参数
- [ ] 日志轮转机制正常工作（按日期或大小轮转）
- [ ] 敏感信息（私钥、API Secret）不被记录到日志

- [ ] 系统能够正确解析价格与时间戳数据
- [ ] WebSocket 断线后能够自动重连（指数退避策略）

### Polymarket RTDS
- [ ] 系统能够成功连接 Polymarket RTDS（wss://ws-live-data.polymarket.com）
- [ ] 系统能够成功订阅 crypto_prices_chainlink 主题的 btc/usd 数据流
- [ ] 系统能够正确解析 Chainlink 预言机价格
- [ ] WebSocket 断线后能够自动重连

### Redis 发布
- [ ] 系统能够每秒发布一次消息至 Redis `market_data` 频道
- [ ] 消息格式正确：`{"timestamp": 1234567890123, "binance_price": 67234.50, "chainlink_price": 67235.20}`

- [ ] 断线期间能够继续发布最近一次有效价格（标记为 stale）

## 模块 2：策略分析引擎验证

### 模块独立运行
- [ ] 模块能够独立启动（python -m modules.strategy_engine.main）
- [ ] 模块崩溃后能够自动重启
- [ ] 模块日志独立记录至 logs/strategy_engine.log

### Redis 订阅
- [ ] 系统能够成功订阅 Redis `market_data` 频道
- [ ] 系统能够正确解析价格数据
- [ ] 系统能够持续接收市场数据更新

### 价格队列管理
- [ ] 队列能够正确维护 180 个数据点
- [ ] 队列满时能够自动移除最旧数据
- [ ] 队列操作线程安全
- [ ] 能够获取最近 N 秒的价格数据

### OLS 线性回归
- [ ] 系统能够对 180 个价格点进行 OLS 线性回归拟合
- [ ] 系统能够正确计算斜率 K（波动子因子）
- [ ] 系统能够正确计算决定系数 R²
- [ ] K > 0 时判断为上涨趋势，K < 0 时判断为下跌趋势
- [ ] OLS 计算延迟 < 10ms

### 动态安全垫
- [ ] 系统能够正确计算 Base Cushion（默认 $15）
- [ ] 系统能够正确计算 Buffer Cushion（α × |K| × √Time Remaining）
- [ ] 系统能够正确计算安全垫总厚度
- [ ] 安全垫厚度随剩余时间衰减
- [ ] 安全垫厚度随波动率（|K|）增加而增厚

### 买入信号生成
- [ ] 系统能够正确计算差额 |Current Price - Start Price|
- [ ] 系统能够正确判断方向（UP 或 DOWN）
- [ ] 差额 > 安全垫总厚度时触发买入信号
- [ ] 信号仅在 T-100s 至 T-10s 时间窗口内生成
- [ ] 系统能够根据剩余时间查表获取最大买入价格（T-90s: $0.75, T-10s: $0.90）

### Redis 发布（交易信号）
- [ ] 系统能够发布买入信号至 Redis `trading_signal` 频道
- [ ] 买入信号格式正确：`{"action": "BUY", "direction": "DOWN", "max_price": 0.85, "timestamp": 1234567890123}`
- [ ] 等待信号格式正确：`{"action": "WAIT", "timestamp": 1234567890123}`

### 市场生命周期管理
- [ ] 系统能够在 T-300s 正确初始化市场（预计算 Slug、获取元数据、清空队列）
- [ ] 系统能够在 T-300s 至 T-101s 持续收集价格数据
- [ ] 系统能够在 T-100s 至 T-0s 每秒执行策略判断
- [ ] 系统能够在 T-0s 后正确记录结果并重置状态
- [ ] 系统能够无缝切换到下一个 5 分钟周期
- [ ] 周期切换时无状态残留

- [ ] p = 0.5 时手续费达到峰值 1.80%

### 价格验证与订单执行

- [ ] 系统能够通过 CLOB API 获取当前最优买入价格
- [ ] 系统能够判断：当前买入价 < 最高可买入价格
- [ ] 判断通过时能够创建 FAK 市场订单（create_and_post_market_order）
- [ ] 订单类型正确设置为 OrderType.FAK
- [ ] 订单价格符合策略计算的最大买入价格
- [ ] 订单金额考虑动态手续费

### 订单簿深度检查
- [ ] 系统能够通过 CLOB API 拉取实时订单簿
- [ ] 系统能够计算从最优卖价往下吃单的真实成交均价
- [ ] 系统能够验证单次下注金额不超过目标价格层级流动性深度的 50%

### 错误处理
- [ ] 订单提交返回错误时能够记录错误日志并放弃当前交易机会
- [ ] 系统能够继续监听下一个交易信号
- [ ] 系统不重试失败的订单（避免逆向选择）
