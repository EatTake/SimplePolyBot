# SimplePolyBot 完整部署与使用百科全书式文档 Spec

## Why

当前项目缺少一份面向非技术用户（新手小白）的完整部署与使用教程。用户需要从零开始在 Vultr Ubuntu 24.04 服务器上部署系统，但现有 README 仅面向开发者，缺乏：

* 逐步式服务器环境搭建教程

* 每个交易策略参数的详细影响说明与实例

* 日常运维操作指南

* 常见问题与异常处理方案

本规格将生成一本**百科全书式的完整说明书**，覆盖从部署到实际使用的每一个环节。

### Requirement: 第 3 章 - 交易策略详解（核心章节）

#### 3.2 安全垫机制 (Safety Cushion) ⭐ 重点

* 公式详解：`Total = Base + α × |K| × √TimeRemaining`

* **每个参数对买入价格的直接影响**（含数值举例）：

| 参数              | 值变化         | 对买入价格的影响      | 举例                                      |
| --------------- | ----------- | ------------- | --------------------------------------- |
| base\_cushion   | 0.02 → 0.05 | 买入价降低 3 分钱    | 当前价 0.50 → 买入价从 0.48 降到 0.45            |
| alpha           | 0.5 → 0.7   | 波动大时多降 40% 缓冲 | K=0.01, Time=60s: buffer 从 0.27 升到 0.38 |
| slope\_k (K)    | 由 OLS 回归得出  | 波动越大缓冲越厚      | K=0.001 → 几乎无影响; K=0.05 → 大幅降低买入价       |
| time\_remaining | 100s → 10s  | 时间越少缓冲越小      | √100=10 vs √10=3.16，buffer 差 3 倍        |

* **实际场景举例**：

  ```
  场景：BTC 5分钟涨跌市场，当前价格 0.52，起始价格 0.50

  参数设置：base_cushion=0.02, alpha=0.5
  计算过程：
  ① 价格差额 = |0.52 - 0.50| = 0.02 ✅ (超过最小阈值 0.01)
  ② OLS 回归得斜率 K = 0.008, R² = 0.72 ✅ (R² > 0.5)
  ③ 剩余时间 = 45 秒 ✅ (在 10-100s 窗口内)
  ④ Base Cushion = 0.02
  ⑤ Buffer Cushion = 0.5 × |0.008| × √45 = 0.5 × 0.008 × 6.71 = 0.027
  ⑥ Total Cushion = 0.02 + 0.027 = 0.047
  ⑦ 最大买入价格 = 0.52 - 0.047 = 0.473 → 取整 0.47

  结论：以不超过 0.47 的价格买入 YES 代币
  ```
